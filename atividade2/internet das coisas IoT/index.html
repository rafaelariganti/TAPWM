Internet das coisas (IoT)

Conceito

Internet das Coisas (IoT) é a tecnologia que permite conectar objetos físicos à internet, permitindo que eles troquem dados.

Os dispositivos IoT são equipados com sensores e software que permitem coletar e transmitir dados Os dados podem ser usados para automatizar ações ou informar os utilizadores A IoT pode ser usada em casa, no carro, nas lojas, na saúde, no agronegócio, entre outros.

Exemplos de aplicações:
Monitorar e controlar a temperatura, a iluminação e os sistemas de segurança de uma casa;
Evitar colisões, planear rotas e estacionar em vagas apertadas;
Monitorizar pacientes remotamente, por exemplo, com relógios inteligentes e dosadores de medicamentos;
Medir a umidade, nutrientes e temperatura do solo para a irrigação e uso de fertilizantes;
Monitorar operações de plantio e colheita, identificar pragas e analisar o crescimento das culturas.

Protocolos utilizados

Os protocolos da Internet das Coisas (IoT) são padrões de comunicação que permitem que os dispositivos troquem dados. Eles são responsáveis por garantir que as informações de um dispositivo sejam lidas e compreendidas por outro.

Exemplos:

MQTT

MQTT (Message Queuing Telemetry Transport) é um protocolo de comunicação entre dispositivos que permite a troca de mensagens entre máquinas. É um dos principais protocolos utilizados na Internet das Coisas (IoT).

Principais características:
É um protocolo leve e otimizado para redes TCP/IP;
É baseado no modelo Publicador-Subscritor;
É econômico em termos de banda de internet;
É seguro, com vários mecanismos de autenticação e segurança de dados;
É fácil de implementar.

Aplicações:
Automação de processos, Controle de mercadorias, Controle de fluxo de pessoas, Controle de eficiência energética, Monitoramento de oleodutos.

CoAP

O protocolo CoAP (Constrained Application Protocol) é um protocolo de internet que permite a comunicação de dispositivos restritos com a internet. Ele é adequado para dispositivos com pouca energia, como sensores, wearables e sistemas embarcados.

Características do CoAP:
Utiliza UDP (User Datagram Protocol) em vez de TCP (Transmission Control Protocol);
Tem um cabeçalho mais leve e uma abordagem binária e não textual;
É inspirado no HTTP e segue princípios de RESTful;
Utiliza os métodos GET, POST, PUT e DELETE;
É altamente confiável, com mecanismos para garantir a entrega de mensagens;
É escalável, podendo ser adaptado a vários dispositivos em diversas aplicações IoT.

AMQP

AMQP (Advanced Message Queuing Protocol) é um protocolo de comunicação que permite a troca de mensagens entre aplicações. Ele é usado para criar aplicativos híbridos que podem ser executados em várias plataformas.

Principais características do AMQP:
É um protocolo padrão aberto;
É orientado para conexão;
Usa uma codificação binária;
Incorpora esquemas de controle de fluxo;
É programável, permitindo que as aplicações definam rotas, exchanges, filas, e outras entidades
Permite a interoperabilidade entre muitas tecnologias e plataformas.

Aplicações do AMQP:
Serviços financeiros, Internet das Coisas (IoT), Cloud computação.

WI-FI

Wi-Fi é uma tecnologia que permite a conexão à internet sem fio, usando ondas de rádio. Foi criada em 1997 e se tornou popular com o surgimento dos primeiros computadores portáteis.

Como funciona
O Wi-Fi usa o padrão IEEE 802.11;
O roteador Wi-Fi conecta-se ao modem da operadora de internet por cabo de rede;
O roteador divide a conexão com outros dispositivos, que podem se conectar por Wi-Fi ou cabo de rede;
Cada roteador cria uma rede própria, com um nome (SSID) e uma senha;
Para se conectar, os dispositivos precisam ter uma placa de rede Wi-Fi embutida.

Bluetooth

Bluetooth é uma tecnologia de comunicação sem fio que permite a troca de dados entre dispositivos digitais. É uma ligação direta entre dois dispositivos, sem a necessidade de uma rede de suporte, como um roteador.

Como funciona
O Bluetooth transmite dados por ondas de rádio de curto alcance, na faixa de 2,4 GHz;
Para que dois dispositivos se conectem, eles precisam suportar o mesmo perfil;
Ao ligar dispositivos Bluetooth pela primeira vez, é necessário registrá-los, o que é chamado de "emparelhamento".

DDS - Data Distribution Service

Protocolo que integra dispositivos inteligentes de alta performance, enquanto a Internet das Coisas (IoT) é uma rede de objetos e dispositivos conectados à internet.

DDS É um protocolo centrado em dados, Integra dispositivos inteligentes de alta performance.

IoT
É uma rede de objetos e dispositivos conectados à internet;
Permite que objetos sejam dotados de dispositivos sensoriais;
Pode ser usada para otimizar processos, reduzir custos e acidentes, e aumentar a sustentabilidade
É amplamente usada em ambientes industriais (IIoT) e é sinônimo da Indústria 4.0.

Aplicações da IoT
Automatização domiciliar;
Monitoramento remoto de pacientes;
Redução de custos e melhoria da gestão organizacional;
Contagem em tempo real do número de visitantes;
Geração de feedback sobre campanhas de marketing e alterações de preços.

M2M

Machine-to-Machine (M2M) é uma tecnologia que permite que máquinas e dispositivos troquem informações automaticamente, sem a necessidade de intervenção humana.

Como funciona
M2M utiliza redes como redes celulares e protocolos de comunicação M2M especializados.
Sensores e medidores podem comunicar informações registradas, como temperatura ou nível de estoque, para software de aplicação.
O software de aplicação pode usar essas informações para ajustar processos ou fazer pedidos de reposição de estoque.

Aplicações
M2M é usado em vários setores, como manufatura, saúde, negócios e seguros.
M2M é a base da internet das coisas (IoT).
M2M pode ser usado para automatizar, otimizar e agilizar processos.
M2M pode ser usado para medir a eficiência de etapas produtivas ou as condições de armazenamento.

Zigbee

Zigbee é uma tecnologia de comunicação sem fio que permite a interconexão de dispositivos. É muito utilizada em automação residencial e industrial.

Características:
Baixo consumo de energia;
Rede de malha para transmissão de dados;
Cobertura mais ampla e eficiente;
Segurança robusta;
Capacidade de suportar grandes redes de dispositivos interconectados;
Facilidade de implementação.

Desafios

A Internet das Coisas (IoT) enfrenta desafios de segurança, infraestrutura, gestão de dados e outros.


Autenticação fraca: Os dispositivos IoT usam senhas padrão e fracas.
Criptografia de dados: Os dispositivos IoT não protegem adequadamente os dados confidenciais.
Software vulnerável: Os dispositivos IoT podem ter software vulnerável, que é difícil de corrigir.
Protocolos inseguros: Os dispositivos IoT usam interfaces e protocolos de rede inseguros.
Falta de padronização: A falta de padrões e requisitos de segurança contribui para a segurança fraca da IoT.

Desafios de infraestrutura
Falta de investimentos: A IoT depende de uma infraestrutura de telecomunicações robusta, mas o Brasil enfrenta problemas de conectividade em muitas regiões.
Desafios de gestão de dados
Gerenciamento de dados: A implementação de chips IoT gera uma quantidade massiva de dados, que precisam ser gerenciados, armazenados e analisados de maneira eficiente.
Outros desafios Má gestão e configuração incorreta, Desconhecimento dos dispositivos conectados, Fragilidade que a IoT traz para um sistema, Ataques cibernéticos.
